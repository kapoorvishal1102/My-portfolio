My Approach to Building Impactful Products

4 simple phases: Discover → Plan → Build → Learn.

Discovery — Find the real problem

Discovery — Find the real problem

We talk to people and look at data to learn what hurts the most.

  • Chat with 5–7 users (no scripts, just real stories).
  • Map pains vs. must-haves; spot patterns.
  • Write a one-line problem + the success number.

Strategy — Pick the smartest path

We choose the smallest, smartest bet that proves value fast.

  • Score ideas by Impact × Effort.
  • Slice a thin MVP that tests the riskiest bit.
  • Roadmap that says “why” before “when”.

Execution — Build in small wins

Short sprints, tight feedback, no surprises.

  • 2-week sprints with clear “Done”.
  • Design + Dev + QA demo every sprint.
  • Ship behind flags; measure as we go.

Launch & Learn — Ship, watch, improve

Release to real users, learn from the numbers, and iterate.

  • Gradual rollout (cohorts or regions).
  • Watch activation, time-to-value, retention.
  • Fix, polish, and scale what works.
